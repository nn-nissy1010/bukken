# CLAUDE.md - マンション大規模修繕 居住者専用サイト デザイン・HTML開発ガイド

## 1. プロジェクト概要とミッション
あなたはプロフェッショナルなWebデザイナー兼フロントエンドエンジニアです。
提供されたワイヤーフレーム（構造設計図）をもとに、居住者が迷わず直感的に使える、信頼感と清潔感のある高品質な静的デザイン案（HTML / CSS / Vanilla JS）を構築してください。

・案件名：◯◯マンション 大規模修繕 居住者専用サイト デザイン開発
・目的：居住者への工事情報・スケジュールの提供、および問い合わせ対応を円滑にする専用ポータルサイトのデザインモックアップ作成。
・将来的な前提：完成したHTMLは、後工程でWordPressマルチサイトの「横展開用テーマ雛形」として組み込みます。そのため、コードは汎用性が高く、保守しやすいシンプルな構造にしてください。

---

## 2. デザイン方針とトンマナ（Tone & Manner）
信頼感・見やすさ・シニア層を含む幅広い住民の方への「分かりやすさ」を最優先とし、落ち着いた配色と広めの余白で仕上げてください。

### カラーパレット（推奨案）
・メインカラー（ベース＆ヘッダー）：ディープネイビー（#1A365D） - 信頼と誠実さを表現
・アクセントカラー（CTA・重要ボタン）：ウォームゴールド・ベージュ（#C5A059 または #D69E2E） - 目立ちつつも落ち着いた印象
・ベース背景色：オフホワイト・ライトグレー（#F8FAFC） - 画面の圧迫感を軽減
・テキストカラー：ダークチャコール（#2D3748） - 真っ黒（#000）を避け、可読性を向上
・ボーダー・区切り線：ソフトグレー（#E2E8F0）

### タイポグラフィと余白
・フォント：ゴシック体ベース（"Noto Sans JP", "Hiragino Sans", "Meiryo", sans-serif）。
・フォントサイズ：本文は最小16px（SPは15px）をキープし、行間（line-height: 1.7〜1.8）を広めにとって可読性を担保。
・余白設計：セクション間のマージンやカードのパディングを十分に確保し、タップしやすく読み疲れしないレイアウトにする。

## 3. 技術仕様とコーディング規約

### 使用技術
・HTML5：セマンティックなタグ設計（header, nav, main, section, article, footerなど）。
・CSS3：外部ファイル `style.css` に集約。CSSカスタムプロパティ（変数）を活用してテーマカラーや余白を管理すること。
・JavaScript：外部ファイル `main.js` に集約。jQueryや外部フレームワークは非使用（Vanilla JSのみで実装）。アコーディオンやタブ切替、SPメニューの開閉を制御。

### レスポンシブ設計
・モバイルファーストで記述し、PC表示（ブレークポイント 768px / 1024px など）を最適化。
・SP表示ではハンバーガーメニュー格納、PC表示ではヘッダー横並びナビゲーションを展開。

### 実装上の注意点（WordPress実装を見据えた考慮）
・ハードコーディングの回避：物件名（◯◯マンション）やロゴ、電話番号などは後からPHPで動的に置換しやすいよう、明確なクラス名（例：`.site-title`, `.tenant-name`）を付与する。
・コンポーネントの再利用性：記事カード、ボタン（CTA）、アコーディオン、バッジなどのパーツは共通クラスで使い回せる設計にする。

## 4. 作成するファイル構造案
プロジェクトのディレクトリ構成は以下のように整理して出力してください。

・index.html：ログイン前トップ（公開ゾーン・1ページ完結）
・contact-public.html：お問い合わせ（ログイン前専用）
・member-top.html：会員ページトップ（ログイン後・住民専用）
・plan.html：工事の計画（会員専用）
・faq.html：Q&A（会員専用）
・contact-member.html：ご意見の窓口／お問い合わせ（入力・確認・完了のUI実装含む）
・news-list.html：お知らせ一覧（アーカイブ・資料DL・動画一覧の共通テンプレート）
・admin-demo.html：管理画面モックアップ（物件基本設定・表示手動切り替えUI）
・assets/css/style.css：全画面共通スタイルシート
・assets/js/main.js：UI挙動制御スクリプト（アコーディオン、メニュー、確認画面切替など）
・assets/img/：プレースホルダー画像やアイコン配置用

## 5. 各ページの具体的な設計要件（ワイヤーフレーム準拠）

### ページ1：ログイン前トップ（index.html）
・メインビジュアルを美しく配置し、「大規模修繕工事の計画状況を、いつでもご確認いただけます」のキャッチコピーを明示。
・「居住者専用ログイン」ボックスを視覚的に目立たせ、ID・パスワード入力欄とログインボタンをわかりやすく配置。
・「はじめての方へ」はVanilla JSによるアコーディオン開閉として実装。
・フッター前にログイン前専用お問い合わせへのCTAを配置。

### ページ2：お問い合わせ（ログイン前）（contact-public.html）
・ステップバー（1 入力 ＞ 2 確認 ＞ 3 完了）を配置。
・入力フォーム：お名前（任意）、部屋番号（任意）、ご連絡先（任意）、お問い合わせ内容（必須）。

### ページ3：会員ページトップ（member-top.html）
・SPはハンバーガーメニュー、PCはヘッダーナビゲーションを実装。
・上部に固定サマリーエリア「工事の計画 ― なぜ大規模修繕を行うのか」のピン留めカードを配置。
・以下の順序でセクションを縦スクロールで配置：
  1. 新着（掲示板）：タブ切替チップ（すべて／お知らせ／スケジュール／工事の計画）と最新記事4件。
  2. 最近の資料：最新3件のPDFダウンロード行レイアウト。
  3. 動画アーカイブ：動画サムネイル2件のグリッド配置（iframe埋め込みを想定したアスペクト比固定コンテナ）。
  4. Q&Aへの導線。
  5. ご意見の窓口へのCTA。

### ページ4：工事の計画（plan.html）
・パンくずリスト（お知らせ ＞ 工事の計画）を配置。
・全物件共通の定型説明文を読みやすく美しいタイポグラフィでレイアウト。
・説明会資料（PDF）のダウンロード一覧セクションを設置。

### ページ5：Q&A（faq.html）
・よくあるご質問を質問枠タップで回答がアニメーション展開するアコーディオン（Vanilla JS）で構築。
・質問内容（費用、スケジュール、生活影響など）をワイヤーフレーム通りに記述。

### ページ6：ご意見の窓口（contact-member.html）
・ステップバーを配置。
・特徴的な要件：ラジオボタンまたはボタン状のセレクタによる「種別（工事のこと／計画・全体のこと／生活・管理のこと／その他）」選択UI。
・JSを使い、送信ボタンタップでページ内で「確認画面」→「完了画面」のモックアップ表示が切り替わる挙動を実装してください。

### ページ7：お知らせ一覧・共通アーカイブ（news-list.html）
・カテゴリ絞り込みチップと、記事一覧、ページネーション（1, 2, 3...）を実装。
・このHTMLはWordPressの「お知らせ一覧」「資料一覧」「動画一覧」の共通骨格となるため、リスト部分をクリーンなマークアップにしておくこと。

### ページ8：管理画面イメージ（admin-demo.html）
・MRC様の管理画面デモとして、各コンテンツ（お知らせ、工事の計画、資料、動画等）の「公開 ON/OFF」「メニューに表示 ON/OFF」をトグルスイッチやラジオボタンで直感的に切り替えられるUIを作成。
